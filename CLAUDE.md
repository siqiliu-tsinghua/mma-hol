# mma-hol — operational notes

Kernel-minimal higher-order logic theorem prover in Wolfram Language, LCF-style, modeled on HOL Light. **stdlib COMPLETE (2026-06-17): ℝ via Dedekind cuts + sequence theory + closed-interval & set compactness + connectedness + supporting point-set topology — all in `stdlib/Real/`, single construction, NO calculus, 0-sorry.** Capstones (all proven): monotone convergence, Cauchy completeness, Bolzano–Weierstrass, Heine–Borel (closed interval + open-cover `isCompact ⟺ closed∧bounded ⟺ sequentiallyCompact`), IVT/connectedness. **Why this scope**: the project's primary purpose is to test whether WL's `Module`/`Unique` gensym mechanisms can give C++/Java-style `private` encapsulation around an LCF trust boundary; the kernel + number tower + automation + the ℝ theory already demonstrate it. (Lebesgue-number lemma, calculus, M9 multivariable/Stokes, M10 Fourier/Radon all CUT — out of scope; Heine–Borel goes via the sup principle, not the countable-basis/Lindelöf route.) The proof routes were guided by a (now-removed, gitignored) Lean blueprint `tautology` and partly translated via Codex. See `docs/dev/PLAN.md` §1 + §7 M8.

**Design document: `docs/dev/PLAN.md`** — authoritative for all architectural questions. This file is the session-startup summary only; per-milestone proof detail lives in git commit messages and code comments.

## Environment

- Wolfram Language 14.3 via `wolframscript` 1.13 (Linux). Notebooks are optional front-end only.
- All library code must be script-executable: `wolframscript foo.wls` — exit 0 pass, nonzero fail. Hard requirement.

## Trust boundary (the one thing to never get wrong)

The **kernel** (types + terms + 10 primitive rules + axioms + `new_{constant,definition,basic_type_definition,axiom}`) owns all `thm` value construction. Anything outside is untrusted. Worst-case bug in untrusted code is "can't prove a true theorem"; it must never produce a false theorem.

Hard rules:

0. **All mutable kernel state — `thmTag`, `arityTable`, `constTypeTable`, `axiomList`, `defnList`, `axiomIntakeOpen`, and any future registries — lives in `HOL`Kernel`Private`*`. Read or write outside `Kernel.wl` is forbidden.** Two modes via `Global` $HOLEncapsulationMode`: **`"Strict"`** (default, CI) hides the six state symbols as Module-local gensyms; **`"Stable"`** (dev/persistence) gives them fixed names so `DumpSave` snapshots survive cold boots — there the boundary is convention + CI lint (any `HOL`Kernel`Private`*` reference outside `Kernel.wl` is a bug). Both modes share one `Kernel.wl` body via the `defineKernel[...]` installer; CI runs the full suite in both. Don't add new mutable kernel state outside `HOL`Kernel`Private`*`.
1. **`new_axiom` must disappear after bootstrap.** `ETA_AX`/`SELECT_AX`/`INFINITY_AX` are declared during init, then the API is withdrawn (`lockAxioms[]`). Adding a new axiom is a user decision, not a Claude decision.
2. **Term construction above the kernel goes through `mkVar`/`mkConst`/`mkComb`/`mkAbs`.** Raw `comb[...]`/`abs[...]` literals bypass type-checking and α-normalization — forbidden outside the kernel module.
3. **Kernel is a singleton.** `makeKernel[]` runs once at load; no reset. To clear state, restart `wolframscript`.
4. **Bound variables use a separate head `bvar[k_Integer, ty]`** (de-Bruijn levels-from-binder): inside any `abs`, the immediately enclosing binder is `bvar[0, ty]`, k levels out is `bvar[k, ty]`. The origin name lives in `abs[bv, body, origin]`'s 3rd slot for the printer only. Free vars (`var[name, ty]`) and bound vars are structurally distinct heads. **α-equivalence is `aconv`, not `===`** (`===` distinguishes origin) — kernel rules comparing up to α must use `aconv`.
5. **Printer correctness is independent of soundness.** A printer bug rendering `⊢ False` as `⊢ True` won't break the kernel but deceives readers — printer needs regression tests.

## Conventions

- **Working language**: converse with the user in Chinese first, English otherwise. Code, identifiers, and commit messages stay in English.
- **File extensions**: libraries `.wl` (`BeginPackage`/`EndPackage`); entry points/tests/demos `.wls`; notebooks `.nb` optional.
- **New-milestone file layout (from M8/ℝ on): milestone = folder, not monolith** — see **PLAN §8.1**. Split by the internal dependency tree (each small file = one defs/lemmas closure with explicit exports/imports); centralize term-builder/unfold vocabulary (folder files share one package context; cross-milestone helpers in a shared vocab module) so relocation stays cheap. Optimize for cheap boundary change, not perfect up-front interfaces. Existing Num/Int/Rat/FTA stay monolithic (no retro-split). The 2026-06 Rat→Num/FTA/Int migration that motivated this is the cautionary tale.
- **Contexts**: `HOL`Kernel``, `HOL`Bool``, … Private helpers in `` `Private` `` subcontext. **Private symbols don't cross files** — a helper/value defined in one file's `` `Private` `` (e.g. `numInductBy`, `numTy`, `plusTm` in Num.wl) is unreachable by bare name elsewhere; reference the full `HOL`Pkg`Private`name` path or redefine locally, else it silently stays unevaluated and trips a "not a theorem" / "mkComb type mismatch" far downstream.
- **Errors**: kernel throws `Throw[Failure["HOLError", <|"tag"->…, "msg"->…|>], holErrorTag]`; top-level `Catch` in `.wls` entries. Tests assert on tag. **Never `Message` + `$Failed`.** Read payload via `f[[2, "tag"]]` (not `f[[1, …]]`). **`holError` is HoldRest**: msg/payload must be a literal String/Association AT the call site — a computed expression (`name <> "…"`, a payload variable) matches no DownValue, so nothing throws and the call returns unevaluated (bit Codex brief-005).
- **Tests**: `tests/harness.wl` gives `assertEq`/`assertTrue`/`assertThrows`/`runTests`/`testExit`. Runners: `tests/run_all.wls` (cold-boot Strict, CI, ~10-15 min); `tests/run_all_stable.wls` (cold-boot Stable, CI sanity); `tests/run_fast.wls [pattern…]` (restore `bootstrap.mx` snapshot, run matching files, ~3 s). `kernel_tests.wl` is skipped from the fast runner.
- **Snapshot dev loop (the frontier workflow — PLAN §8.1.6, built 2026-06-04).** `bootstrap.mx` = the current *frozen base* = exactly the files in `build_snapshot.wls`'s load list. While developing a NEW file, keep it OUT of that list and iterate with **`tests/dev.wls <frontier.wl> [pat…]`** — restores the base + loads ONLY the frontier file on top + runs matching tests (~0.1 s vs a ~7-min cold rebuild; staleness-checks all sources EXCEPT the named frontier files). When the file graduates: add it to the three runners' lists (below), then **`tests/extend_snapshot.wls <file.wl>`** folds it into `bootstrap.mx` incrementally (~0.2 s, NOT a cold rebuild). Full **`tests/build_snapshot.wls`** (optional `<uptoPattern>` = partial build through a layer; prints per-file timing) is the cold re-baseline — needed only when an UPSTREAM file changes or to produce a clean authoritative base. dev/extend are Stable-mode like run_fast (catch logic errors, NOT Strict gensym/load-order) → the per-stage authoritative gate stays the cold Strict `run_all`; re-baseline with full `build_snapshot` periodically (extended snapshots can drift a few KB from cold).
- **Adding a module**: add it to the load lists in all three runners — `tests/run_all.wls`, `tests/run_all_stable.wls`, and `tests/build_snapshot.wls` (dependency order). Forgetting `run_all_stable.wls` is how the Stable suite silently drifts out of sync with Strict. (`dev.wls`/`extend_snapshot.wls` need no list — they operate on the snapshot incrementally.) Test files use `Needs["HOL`…`"]` which is a no-op once loaded but can't locate flat-layout files on its own.
- **No comments describing what code does** — names carry that. Comments only for non-obvious *why*.
- **No `_` in identifiers**: WL parses `FOO_BAR` as `Pattern[FOO, Blank[BAR]]`, silently introducing separate symbols that can shadow imports. Applies to Module locals too (`x_eq` → `xEq`). Tactics camelCase (`conjTac`, `asmSimp`); combinators ALL-CAPS underscore-free (`THEN`, `REPEAT`, `SIMP`, `MESON`).
- **No `Function[{_}, body]`**: `_` is not a valid Function parameter; WL emits `Function::flpar`, leaves the call unevaluated, and garbage propagates into kernel functions. Use a real symbol.
- **WL comments close at the first `*)`** — `(a*)`, `(SUC k*)` etc. in comment prose silently terminate the comment; error points far below the cause.
- **Variable-capture hygiene.** A HOL variable's identity is **(name string, type)**, NOT the WL symbol holding it — `Module` seals WL symbols (`qV`/`qBig`) but NOT the `mkVar` *name* (`mkVar["q",…]` in two places = one HOL variable; this caused the `nnMulCutIsCut` `ABS: binder occurs free in hypotheses`). Two **locally-bounded** rules (never "scan all callers"):
  - **(a) Extract, don't rebuild.** A subterm that must α-match kernel output (β-reduced ∃-body, the `IS_CUT` 4-conjunction, …) → take it from the actual theorem (`concl[…][[…]]`), don't hand-reconstruct. The kernel α-renames binders when the substituted term shares their names; a hand-built term won't, and `EQMP`/`EXISTS` reject it on `aconv`. (The `realNnMul` open-case bug: midpoint `½(q+p·q)` contained `p,q`, so a rebuilt `∃p∃q` captured them.)
  - **(b) Distinctive internal binders.** Any term-accepting helper's internal scaffolding binder, and per-branch witnesses in multi-case proofs, get a distinctive name (`funcExtThm`→`xFE`, `negExTerm`→`rNE`) — never a bare common name (`x/q/r/p/s`) a caller's term might carry. Bare short names only for a proof's single widest-scope role + final-statement canonical binders (those are `aconv`/capture-avoiding-substitution protected). The binder-vs-arg **type** difference (real arg / rat binder) often saves you — don't rely on it. **`freshName`/`collectFreeNames` are Bool-private ⇒ unusable in stdlib; use the distinctive-name discipline there.**
- **Resource cost vs convenience**: on hot paths (anything per-node in proof traversal — `freesIn`, `vsubst`, `instType`, `betaSubst`, `tyvarsInTerm`, `aconv`) prefer the costlier-to-write but cheaper-to-run alternative. (Worked precedent: structural `bvar[k, ty]` head over name-encoded indices — ~2× faster per traversal at the price of a `bvar` clause in every term walker.)
- **Debug with narrow probes, never whole-term dumps.** When a kernel rule fails mid-proof, do NOT `Print` the full term tree (`Print[concl[th]]` / `Print[…Short[#,5]&]` on a deep term) — one such dump is thousands of chars, and every later turn re-pays for it in context. Instead probe narrowly: `Head[th]` (is it a `thmTag` or an unevaluated call?), `hyp[th]`, `MatchQ[concl[th], <small pattern>]`, or print a single sub-slot (`concl[th][[1,2]]`). Binary-search the failing step by moving a single shape check, not by dumping everything. Token cost is dominated by accumulated context, so a few KB of term-tree spam per build round compounds fast. (Snapshot rebuilds are wall-clock-heavy but token-cheap; the expensive thing is large tool output.)
